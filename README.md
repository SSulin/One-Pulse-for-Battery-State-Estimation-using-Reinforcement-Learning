# One-Pulse-for-Battery-State-Estimation-using-Reinforcement-Learning

This repository contains datasets, processed features, results, and code for **retired battery information extraction** (including material type, SOC, and SOH), with a focus on **data density, reinforcement learning, and data sufficiency**.

The project is based on a dataset generated by Tsinghua University, incorporating pulse-based feature extraction and XGBoost-based **classification and estimation models**.

---

## Repository Structure

### 1. Pulse Battery Datasets

We conducted physical testing on **270 retired lithium-ion batteries**, covering:

- 3 cathode chemistries (LFP, LMO, NMC)  
- 4 historical usage conditions  
- 3 physical formats  
- 4 nominal capacity designs  

Specifically, the dataset includes:

- 35Ah LFP  
- 10Ah LMO  
- 2.1Ah NMC batteries  

The batteries were tested under diverse operating conditions:

- **State of Charge (SOC):** 5% to 95%  
- **State of Health (SOH):** 0.5 to 1.0  

Unlike conventional constant-current constant-voltage (CCCV) protocols, a **pulse-based testing strategy** was adopted. Each battery was subjected to a sequence of five current pulses with increasing magnitudes:

0.5C → 1C → 1.5C → 2C → 2.5C

### 2. Processed Data

- `NMC_2.1Ah_W_5000.xlsx`  
  Extracted features from 2.1Ah NMC batteries with a pulse width of 5 s. Features are derived from characteristic inflection points, resulting in **20 physically meaningful descriptors**.

- `LMO_10Ah_W_5000.xlsx`  
  Extracted features from 10Ah LMO batteries with a pulse width of 5 s. Features are derived from characteristic inflection points, resulting in **20 physically meaningful descriptors**.

- `NMC_15Ah_W_5000.xlsx`  
  Extracted features from 15Ah NMC batteries with a pulse width of 5 s. Features are derived from characteristic inflection points, resulting in **20 physically meaningful descriptors**.
